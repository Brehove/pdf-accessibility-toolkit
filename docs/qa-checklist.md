# QA Checklist (Higher Ed)

Use this checklist after conversion.

## 1) Structure and navigation
- Confirm heading hierarchy is logical (`H1 -> H2 -> H3`) with no skipped levels.
- Confirm major sections are discoverable using Word's navigation pane.
- Confirm lists are true Word lists (not typed bullets or manual numbering).

## 2) Images and non-text content
- Confirm meaningful images have useful alt text.
- Confirm decorative images use minimal or empty alt text.
- For charts/diagrams, ensure alt text explains the key takeaway, not pixel-level details.

## 3) Tables
- Confirm first row is marked as header on every data table.
- Confirm table reading order is left-to-right, top-to-bottom.
- Confirm merged cells do not break meaning.

## 4) Reading order and page flow
- Confirm paragraph order matches intended reading sequence.
- Confirm page breaks do not split critical heading/content relationships.
- Spot-check OCR output for dropped lines or duplicated lines.

## 5) Metadata and delivery
- Confirm document title is meaningful for LMS and assistive tech.
- Confirm filename is course-friendly and semester-friendly.
- Record known manual edits needed before publishing.
